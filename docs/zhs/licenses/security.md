# 安全策略

## 报告漏洞

**请勿针对安全漏洞公开提 issue。**

请通过
[GitHub 安全公告](https://github.com/celestia-island/arona/security/advisories/new)
私下报告。
如果您无法使用 GitHub 安全公告，请发送邮件至维护者
security@celestia.world，并附上清晰的描述与复现步骤。

## 范围

在范围内：

- 身份验证绕过、JWT/OAuth 弱点、会话处理缺陷
- API 密钥 / 凭据泄露或存储不当
- 授权与 RBAC 实施缺口
- 注入类漏洞（SQL、命令、SSRF、XSS）
- 不安全的反序列化、路径穿越、SSRF
- 允许提权或跨租户访问的问题

不在范围内：

- 上游依赖中无法通过本项目利用的漏洞
- 自托管部署未按文档指引进行安全配置
- 针对公共 LLM 提供商端点的拒绝服务攻击

## 响应

| 阶段 | 目标 |
| --- | --- |
| Agent 确认收到 | 10 分钟内 |
| 真人确认收到 | 1 个日历日内 |
| 初始评估 | 3 个日历日内 |
| 修复或缓解 | 30 个日历日（视严重程度而定） |

请在报告中包含：（1）受影响的组件及版本；（2）攻击向量与影响；（3）复现步骤；（4）建议的缓解措施。

## 支持的版本

仅 `main` / `dev` 分支上的最新发布线会获得安全修复。
