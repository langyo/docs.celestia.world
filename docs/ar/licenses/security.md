# سياسة الأمان

## الإبلاغ عن ثغرة أمنية

**لا تفتح issues عامة للثغرات الأمنية.**

أبلغ عنها بشكل خاص عبر
[GitHub Security Advisories](https://github.com/celestia-island/arona/security/advisories/new).
إذا لم يكن GitHub Security Advisories متاحًا لك، أرسل بريدًا إلكترونيًا إلى المشرف على
security@celestia.world مع وصف واضح وخطوات إعادة الإنتاج.

## النطاق

ضمن النطاق:

- تجاوز المصادقة، نقاط ضعف JWT/OAuth، عيوب معالجة الجلسة
- إفصاح مفاتيح API / بيانات الاعتماد أو التخزين غير السليم
- ثغرات التفويض وتطبيق RBAC
- ثغرات الحقن (SQL، الأوامر، SSRF، XSS)
- إلغاء التسلسل غير الآمن، اجتياز المسار، SSRF
- مشكلات تسمح بتصعيد الامتيازات أو الوصول بين المستأجرين

خارج النطاق:

- الثغرات في تبعيات المنبع غير القابلة للاستغلال عبر هذا المشروع
- عمليات النشر المستضافة ذاتيًا بتكوين غير آمن يخالف الإرشادات الموثقة
- هجمات حجب الخدمة ضد نقاط نهاية مزود LLM العام

## الاستجابة

| المرحلة | الهدف |
| --- | --- |
| إقرار استلام الـ Agent | 10 دقائق |
| إقرار استلام بشري | 1 يوم تقويمي |
| التقييم الأولي | 3 أيام تقويمية |
| الإصلاح أو التخفيف | 30 يومًا تقويميًا (حسب درجة الخطورة) |

يرجى تضمين: (1) المكوّن والإصدار المتأثران، (2) متجه الهجوم والأثر، (3) خطوات إعادة الإنتاج، (4) التدابير التخفيفية المقترحة.

## الإصدارات المدعومة

فقط أحدث خط إصدار على الفرعين `main` / `dev` يتلقى إصلاحات أمنية.
