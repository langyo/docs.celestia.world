# Пул SSH-подключений

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Исходный код открывал новое SSH-подключение для каждой операции (список файлов, выполнение команды). Это затратно (TCP + обмен ключами + аутентификация на каждый вызов) и не масштабируется для интерактивного использования с несколькими параллельными операциями.

## Решение

Реализовать `SshConnectionPool`, индексированный по `(host, port, username)`. Подключения устанавливаются лениво при первом запросе и переиспользуются между операциями. `PooledSshClient` оборачивает разделяемый `Arc<Mutex<SshSession>>`. Периодические проверки работоспособности удаляют мёртвые подключения.

## Последствия

### Положительные

- Значительное снижение задержки для повторяющихся операций.
- Позволяет параллельные сеансы оболочки, файлов и терминала через одно подключение.

### Отрицательные

- Пул должен обрабатывать истечение срока действия и переподключение.
- Долгоживущие подключения могут быть разорваны межсетевыми экранами.
