# Транспорт сигнализации — двойной Unix-сокет / TCP

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Клиент сигнализации подключается к серверу ретрансляции для обмена SDP-предложениями/ответами WebRTC и кандидатами ICE. Изначально использовались только Unix-доменные сокеты (`tokio::net::UnixStream`), которые недоступны на Windows. Для кроссплатформенной поддержки требуется резервный вариант.

## Решение

Реализовать двухтранспортное перечисление `SignalingStream`, которое сначала пробует Unix-доменный сокет (на платформах, которые его поддерживают, определяется по формату пути — начинается с `/` или заканчивается на `.sock`), с откатом на TCP. Адаптер `TransportWriter` абстрагирует `AsyncWrite` для обоих транспортов. На платформах, отличных от Unix, доступен только TCP.

## Последствия

### Положительные

- Кроссплатформенная сигнализация; один протокол JSON-RPC поверх обоих транспортов

### Отрицательные

- Сигнализация по TCP не зашифрована по умолчанию
- Пользователи Windows должны обеспечить привязку только к loopback
