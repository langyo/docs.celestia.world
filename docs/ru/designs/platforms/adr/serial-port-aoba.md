# Последовательный порт через aoba

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Evernight требуется поддержка последовательных портов для управления встраиваемыми устройствами и зондирования промышленного протокола (Modbus RTU). Родственный крейт `aoba` уже предоставляет кроссплатформенное перечисление последовательных портов, извлечение VID/PID/серийного номера и функциональность мастера Modbus RTU/TCP.

## Решение

Делегировать все операции последовательного порта крейту aoba. Модуль `serial` в Evernight определяет типы (`SerialConfig`, `SerialPortInfo`) и реализует `TerminalBackend` поверх последовательного транспорта, вызывая aoba для фактического ввода-вывода порта. Автоопределение протокола (сканирование скорости/чётности Modbus RTU) также делегируется aoba.

## Последствия

### Положительные

- Повторное использование проверенного кода; aoba обрабатывает кроссплатформенные краевые случаи.

### Отрицательные

- Добавляет зависимость от aoba; функциональность serial требует доступности aoba.
