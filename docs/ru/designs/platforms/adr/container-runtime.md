# Клиент контейнерной среды выполнения (Docker/Podman)

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Управление контейнерами (Docker, Podman) — ключевой сценарий использования универсального менеджера подключений. Операции включают: список контейнеров, запуск оболочки через exec, просмотр логов и проброс портов.

## Решение

Использовать Docker Engine API через Unix-сокет (или named pipe в Windows). Совместимый с Docker API Podman поддерживается через тот же код. Определить типизированные модели контейнеров (`ContainerInfo`, `ContainerState`, `ContainerPort`). В будущем: реализовать `TerminalBackend` поверх Docker exec attach для интерактивных оболочек контейнеров.

## Последствия

### Положительные

- Docker API хорошо документирован и стабилен; совместимость с Podman бесплатна.

### Отрицательные

- Требуется запущенный демон Docker/Podman.
- Различия в версиях API между версиями Docker.
