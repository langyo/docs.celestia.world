# Обработка ошибок — thiserror с крейтовым Result

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Evernight требуется единый тип ошибок для всех модулей (screen, SSH, hardware, networking, tunneling). Библиотека должна предоставлять чистый API ошибок, сохраняя эргономичность внутренней обработки ошибок.

## Решение

Использовать `thiserror` для создания перечисления `EvernightError` с реализациями отображения `#[error(...)]` для каждого варианта. Определить `pub type Result<T> = std::result::Result<T, EvernightError>` как тип результата уровня всего крейта. Каждый вариант фиксирует доменно-специфичный контекст (ScreenCapture, Ssh, Tunnel и т.д.) в виде String, а не оборачивает внешние типы ошибок, чтобы избежать утечки деталей внутренних зависимостей.

## Последствия

### Положительные

- Стабильный публичный API ошибок; потребители сопоставляют варианты перечисления, не зная внутренних деталей

### Отрицательные

- Некоторая потеря информации из-за преобразования в `String`
- Отсутствие структурированной цепочки ошибок
