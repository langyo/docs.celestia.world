# Бэкенд SSH — russh с разделяемым подключением

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Evernight требуется SSH для удалённой оболочки, файловых операций и терминального доступа. Изначально кодовая база имела три отдельные реализации обработчика SSH (FileHandler, SshHandler, TerminalHandler) с дублированной логикой подключения. Проект предписывает использование чистого Rust (без вызовов оболочки `Command::new("ssh")`).

## Решение

Использовать `russh` (чистая реализация SSH-2 на Rust) в качестве бэкенда SSH. Объединить все реализации обработчиков SSH в единый `DefaultSshHandler` в `remote/connection.rs` с общей функцией `connect_session()`. Все операции SSH (оболочка, файлы, терминал) используют эту общую абстракцию подключения.

## Последствия

### Положительные

- Единый источник истины для логики аутентификации SSH
- Легко добавить пул подключений в будущем

### Отрицательные

- `russh` может отставать от OpenSSH в краевых случаях
- Пока нет встроенной пересылки SSH-агента
