# Абстракции типажей TerminalBackend / ViewportBackend / FileBackend

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Evernight требуются полиморфные интерфейсы бэкендов, чтобы фронтенды (CLI, TUI, GUI) могли единообразно взаимодействовать с любым протоколом. Без общих типажей каждому фронтенду потребовались бы отдельные пути кода для каждого протокола: терминального, графического и файлового.

## Решение

Определить три объектно-безопасных асинхронных типажа в корне крейта (всегда доступны, без feature-флагов):

- **`TerminalBackend`** — `read` / `write` / `resize` / `close`
- **`ViewportBackend`** — `render` / `input` / `clipboard` / `close`
- **`FileBackend`** — `list` / `stat` / `get` / `put` / `rm` / `mkdir` / `rename`

Каждый протокольный бэкенд реализует соответствующие типажи. Фронтенды используют `Box<dyn TerminalBackend>` и т.д.

## Последствия

### Положительные

- Фронтенды не зависят от протокола; новые бэкенды (например, RDP) подключаются без изменений во фронтендах.

### Отрицательные

- Асинхронные типажи-объекты требуют накладных расходов `Box::pin`.
- Дизайн типажей должен быть стабильным, так как его изменение ломает все реализации.
