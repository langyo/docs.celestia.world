# Парсер конфигурации SSH

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Пользователи ожидают, что `evernight` будет читать `~/.ssh/config` для псевдонимов хостов, jump-хостов и файлов ключей так же, как команда `ssh`. Без этого пользователям приходится повторять все параметры подключения при каждом вызове.

## Решение

Реализовать парсер конфигурации SSH на чистом Rust, который обрабатывает стандартные директивы: `Host`, `HostName`, `User`, `Port`, `IdentityFile`, `ProxyJump`, `ForwardAgent`, `ServerAliveInterval` и т.д. Сопоставление glob-шаблонов для записей `Host`. Интегрирован с пулом подключений для прозрачных подключений на основе конфигурации.

## Последствия

### Положительные

- Совместимость с SSH «из коробки»; разрешение jump-хостов из конфигурации.

### Отрицательные

- Необходимо отслеживать изменения синтаксиса конфигурации OpenSSH.
- Поддержка директивы `Match` сложна.
