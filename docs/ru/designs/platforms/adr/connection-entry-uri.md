# Схема URI для элемента подключения

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Каталог подключений нуждается в единообразном способе представления различных протокольных соединений (SSH, VNC, RDP, serial, Docker). Схема URI предоставляет человекочитаемые, сериализуемые дескрипторы.

## Решение

Использовать протокольно-специфичные схемы URI:

- `ssh://user@host:port`
- `vnc://host:5900`
- `rdp://host:3389`
- `serial:///dev/ttyUSB0?baud=9600`
- `docker:///var/run/docker.sock?container=name`

`ConnectionEntry` разбирает URI в типизированные структуры с полями scheme, host, port, username, path и параметрами запроса. Каталог представляет собой дерево узлов `ConnectionCategory`, содержащих элементы.

## Последствия

### Положительные

- Знакомый формат URI; легко сериализуется; поддерживает копирование и передачу подключений.

### Отрицательные

- Некоторые протоколы плохо отображаются на URI (например, контекст Kubernetes).
- Параметры строки запроса неструктурированы.
