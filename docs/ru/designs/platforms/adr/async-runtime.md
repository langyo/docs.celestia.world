# Асинхронная среда выполнения — tokio

- **Статус**: Принято
- **Дата**: 2026-06-09
- **Авторы**: Evernight Core Team

## Контекст

Evernight требуется асинхронная среда выполнения для сетевого ввода-вывода (SSH, WebRTC, TCP-туннели), операций по таймеру (циклы захвата кадров, таймауты протоколов) и управления конкурентными задачами. Экосистема Rust предлагает tokio, async-std и smol.

## Решение

Использовать `tokio` как единственную асинхронную среду выполнения. tokio является стандартом де-факто в асинхронной экосистеме Rust с самой богатой поддержкой драйверов. Ключевые зависимости (russh, webrtc, reqwest) уже требуют tokio. Использовать `tokio::runtime::Handle::current()` для порождения задач из синхронного контекста.

## Последствия

### Положительные

- Совместимость с экосистемой; не требуется мостов между средами выполнения

### Отрицательные

- tokio — тяжёлая зависимость
- Невозможно использовать крейты async-std или smol, не поддерживающие tokio
