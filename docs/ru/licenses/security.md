# Политика безопасности

## Сообщение об уязвимости

**Не создавайте публичные issues для уязвимостей безопасности.**

Сообщайте о них в частном порядке через
[GitHub Security Advisories](https://github.com/celestia-island/arona/security/advisories/new).
Если GitHub Security Advisories вам недоступен, напишите мейнтейнеру на
security@celestia.world с понятным описанием и шагами воспроизведения.

## Область действия

В области действия:

- Обход аутентификации, слабые места JWT/OAuth, недостатки обработки сессий
- Утечка API-ключей / учётных данных или ненадлежащее хранение
- Пробелы в авторизации и применении RBAC
- Уязвимости внедрения (SQL, командная строка, SSRF, XSS)
- Небезопасная десериализация, обход пути, SSRF
- Проблемы, допускающие повышение привилегий или межтенантный доступ

Вне области действия:

- Уязвимости в восходящих зависимостях, не эксплуатируемые через этот проект
- Self-hosted-развертывания с небезопасной конфигурацией вопреки задокументированным рекомендациям
- DoS-атаки на публичные конечные точки LLM-провайдера

## Реакция

| Этап | Целевое время |
| --- | --- |
| Подтверждение получения Agent | 10 минут |
| Подтверждение получения человеком | 1 календарный день |
| Первичная оценка | 3 календарных дня |
| Исправление или смягчение | 30 календарных дней (зависит от серьёзности) |

Пожалуйста, укажите: (1) затронутый компонент и версию, (2) вектор атаки и последствия, (3) шаги воспроизведения и (4) предлагаемые меры смягчения.

## Поддерживаемые версии

Только последняя линия релиза на ветках `main` / `dev` получает исправления безопасности.
