# Evernight 기여 가이드

> 이 파일은 기여 정책의 한국어 버전입니다. 빌드 명령어와 상세 설정 절차는 저장소 루트의 영문
> [`CONTRIBUTING.md`](../../../CONTRIBUTING.md)를 참조하십시오. 명령어 자체는 번역하지 않습니다.
> 충돌이 있을 경우 영문 버전이 우선합니다.

## 기여 정책 (먼저 읽어 주세요)

Evernight는 물리·산업 시스템을 구동할 수 있으므로 **안정성과 안전이 기여 처리량보다 우선합니다**.
Pull Request를 열기 전에 이 절을 먼저 읽어 주세요.

- **병합 기준이 높고, 공개 로드맵이 아닙니다.** PR을 열었다고 해서 병합된다는 뜻은 아닙니다.
  아키텍처에 부합하고 리뷰를 통과한, 의도적으로 소수의 변경만 받아들입니다. 이는 설계상의 선택이지 무례함이 아닙니다.
- **환영하는 것:** 버그 보고, 초점이 맞춘 수정, **주변부**(Layer 3 플러그인, 장치 profile,
  LLM provider 어댑터, 통합, 문서)에 대한 범위가 명확한 개선, 코드 작성 전의 설계 논의.
- **일반적으로 병합하지 않는 것:** 사전 제안 없는 대규모 재작성, 사전 설계 논의 없는 아키텍처 변경,
  대량의 "vibe-coded" PR, 코어의 안전/정확성 기준을 낮추는 모든 것, 초대와 연장된 리뷰 없이
  보안 중요 코어에 대한 변경.
- **코어 vs 주변부.** 코어(오케스트레이션, 마이크로커널, 보안)는 가장 엄격한 기준을 유지하며
  주로 코어 팀이 관리합니다. 외부 기여가 가장 유용하고 수용될 가능성이 높은 곳은 주변부입니다.
- **CLA 필수.** 수용되는 모든 기여는 서명된 CLA가 필요합니다. [`CLA.md`](../../../CLA.md)를 참조하세요.
  커밋에는 `Signed-off-by`(`git commit -s`)가 포함되어야 합니다.

> **라이선스는 열리지만, 병합 기준은 변하지 않습니다.** **2030-01-01**에 이 프로젝트는 BUSL-1.1에서
> Apache-2.0 또는 MIT(수령자 선택)로 전환합니다([`LICENSE`](../../../LICENSE) 참조). 이는 *코드로 할 수 있는 일*을
> 넓히는 것이지, 리뷰 기준을 낮추거나 CLA를 제거하거나 더 많은 PR을 수용한다는 뜻이 아닙니다. 변경일 전후로 기여 정책은 동일합니다.

## 보안

보안 취약점을 공개 이슈로 보고**하지 마세요**.
[GitHub Security Advisories](https://github.com/celestia-island/evernight/security/advisories/new)를 통해 비공개로 보고하세요.
위협 모델과 대응 SLA는 [`SECURITY.md`](../../../SECURITY.md)를 참조하세요.

## 행동 강령

존중하고 건설적이며 포용적으로 행동하세요. [Contributor Covenant Code of Conduct](../../../CODE_OF_CONDUCT.md)를 따릅니다.

## Pull Request 절차

1. Fork하고 `main`에서 분기합니다.
2. 큰 변경은 먼저 issue에서 논의합니다.
3. 원자적 커밋, Conventional Commits를 준수합니다.
4. `just ci`(또는 저장소의 CI 명령)가 통과하는지 확인합니다.
5. CLA에 서명하고 `Signed-off-by`를 추가합니다.
6. 리뷰 의견에 대응합니다. force-push는 rebase에만 사용합니다.

## 라이선스와 CLA

**BUSL-1.1**을 채택하며, **Change Date는 2030-01-01**, 그 시점에 수령자 선택으로 **Apache-2.0 또는 MIT**로 전환합니다.
내부 운영, 학술, 정부, 교육, 비상업적 목적에 대해서는 오늘 이미 Apache-2.0 또는 MIT와 동등합니다
([`LICENSE`](../../../LICENSE)의 Additional Use Grant 참조). 제한되는 상업적 용도(호스팅, 재판매, 서비스로의 리브랜딩 판매)는
Change Date 이전에 별도의 상업 라이선스가 필요합니다.

기여함으로써 귀하는 자신의 기여가 본 프로젝트의 라이선스로 라이선스되는 것에 동의하고, CLA([`CLA.md`](../../../CLA.md))에 서명합니다.
CLA는 프로젝트에 **재라이선스 권한을 포함한** 관대한 라이선스를 부여하여, 프로젝트가 BUSL→Apache/MIT 경로를 유지하고
향후 라이선스를 조정할 수 있게 합니다.
