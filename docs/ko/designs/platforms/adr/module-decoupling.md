# 모듈 분리와 타입 소유권

- **상태**: Accepted
- **날짜**: 2026-06-09
- **작성자**: Evernight Core Team

## 배경

원래 코드베이스는 모든 데이터 타입을 단일 `types.rs` 모놀리스에 배치하여 화면, 하드웨어, 원격, 프로토콜, 터널 타입을 혼합했다. 이는 순환 의존성 위험을 초래했고 어떤 모듈이 어떤 타입을 "소유"하는지 불분명하게 만들었다. 새로운 기능(예: VNC)을 추가하려면 공유 타입 파일을 수정해야 했다.

## 결정

각 도메인의 타입을 해당 모듈의 `types.rs` 파일(예: `screen/types.rs`, `hardware/types.rs`)로 이동한다. 루트 `types.rs`와 `prelude.rs`는 하위 호환성을 위해 모든 타입을 재수출한다. 각 모듈은 자신의 타입에 대한 유일한 소유자이며 독립적으로 진화할 수 있다.

## 결과

### 긍정적 측면

- 명확한 타입 소유권; 교차 모듈 타입 충돌 없음
- 새 모듈이 기존 파일에 영향을 주지 않음

### 부정적 측면

- 소비자는 올바른 모듈에서 임포트하거나 prelude를 사용해야 함
- 재수출 계층은 간접 계층을 추가함
