# 보안 정책

## 취약점 신고

**보안 취약점에 대해 공개 이슈를 열지 마세요.**

[GitHub Security Advisories](https://github.com/celestia-island/arona/security/advisories/new)
를 통해 비공개로 신고해 주세요.
GitHub Security Advisories를 이용할 수 없는 경우, 명확한 설명과 재현 단계를 포함하여
유지관리자(security@celestia.world)에게 이메일을 보내주세요.

## 범위

범위 내:

- 인증 우회, JWT/OAuth 취약점, 세션 처리 결함
- API 키 / 자격 증명 노출 또는 부적절한 저장
- 인가 및 RBAC 적용 누락
- 인젝션 취약점(SQL, 명령, SSRF, XSS)
- 안전하지 않은 역직렬화, 경로 탐색, SSRF
- 권한 상승 또는 테넌트 간 액세스를 허용하는 문제

범위 외:

- 본 프로젝트를 통해 악용할 수 없는 상위 의존성의 취약점
- 문서화된 지침에 반하는 안전하지 않은 구성의 자체 호스팅 배포
- 공용 LLM 제공자 엔드포인트에 대한 서비스 거부 공격

## 대응

| 단계 | 목표 |
| --- | --- |
| Agent 접수 확인 | 10분 이내 |
| 담당자 접수 확인 | 1 역일 이내 |
| 초기 평가 | 3 역일 이내 |
| 수정 또는 완화 | 30 역일(심각도에 따라) |

보고서에 다음을 포함해 주세요: (1) 영향받는 컴포넌트와 버전, (2) 공격 벡터와 영향, (3) 재현 단계, (4) 제안하는 완화 조치.

## 지원 버전

`main` / `dev` 브랜치의 최신 릴리스 라인만 보안 수정을 받습니다.
