# Evernight へのコントリビュート

> これはコントリビューションポリシーの日本語版です。ビルドコマンドと詳細なセットアップ手順は
> リポジトリルートの英語版 [`CONTRIBUTING.md`](../../../CONTRIBUTING.md) を参照してください。
> コマンドは翻訳しません。矛盾する場合は英語版が優先されます。

## コントリビューションポリシー（まずお読みください）

Evernight は物理・産業システムを駆動する可能性があるため、**安定性と安全性がコントリ
ビューションの量より優先されます**。Pull Request を開く前に本節をお読みください。

- **マージ基準は高く、公開ロードマップではありません。** PR を開いてもマージされるとは限りません。
  アーキテクチャに適合しレビューを通過した、意図的に少数の変更のみを受け入れます。これは設計上の
  選択であり、無礼ではありません。
- **歓迎するもの：** バグ報告、焦点を絞った修正、**周辺部**（Layer 3 プラグイン、デバイス
  profile、LLM provider アダプタ、統合、ドキュメント）への範囲が明確な改善、コードを書く前の
  設計議論。
- **通常マージしないもの：** 大規模な事前相談のない書き直し、設計議論を経ないアーキテクチャ変更、
  大量の「vibe-coded」PR、コアの安全性・正確性の基準を下げるもの、招待と拡張レビューのない
  セキュリティクリティカルなコア
  への変更。
- **コア vs 周辺部。** コア（オーケストレーション、マイクロカーネル、セキュリティ）は最も厳しい基準を
  維持し、主にコアチームが保守します。外部の貢献が最も有用で受け入れられやすいのは周辺部です。
- **CLA が必須。** 受け入れられるすべてのコントリビューションには署名済み CLA が必要です。
  [`CLA.md`](../../../CLA.md) を参照してください。コミットには `Signed-off-by`（`git commit -s`）が必要です。

> **ライセンスは開かれますが、マージ基準は変わりません。** **2030-01-01** に本プロジェクトは
> BUSL-1.1 から Apache-2.0 または MIT（受領者の選択）に転換します（[`LICENSE`](../../../LICENSE) 参照）。
> これは*コードで何ができるか*を広げるものであり、レビュー基準を下げたり CLA を撤廃したり、より多くの
> PR を受け入れることを意味しません。変更日の前後を通じてコントリビューションポリシーは不変です。

## セキュリティ

セキュリティ脆弱性を公開 issue で報告**しないでください**。
[GitHub Security Advisories](https://github.com/celestia-island/evernight/security/advisories/new)
から非公開で報告してください。脅威モデルと対応 SLA は [`SECURITY.md`](../../../SECURITY.md) を参照してください。

## 行動規範

尊重と思いやり、建設的かつ包括的であってください。[Contributor Covenant Code of Conduct](../../../CODE_OF_CONDUCT.md) に従います。

## Pull Request の流れ

1. Fork し、`main` から分岐します。
2. 大きな変更は事前に issue で議論します。
3. 原子的なコミットを心がけ、Conventional Commits に従います。
4. `just ci`（またはリポジトリの CI コマンド）が通ることを確認します。
5. CLA に署名し `Signed-off-by` を付与します。
6. レビュー指摘に対応します。force-push は rebase のみにします。

## ライセンスと CLA

**BUSL-1.1** を採用し、**Change Date は 2030-01-01**、その時点で受領者の選択により
**Apache-2.0 または MIT** に転換します。内部運用、学術、政府、教育、非商用目的については、
今日すでに Apache-2.0 または MIT と同等です（[`LICENSE`](../../../LICENSE) の Additional Use Grant 参照）。
制限される商用利用（ホスティング、転売、サービスとしてのリブランド販売）は Change Date まで別途商用ライセンスが必要です。

コントリビュートすることで、あなたは自身の貢献が本プロジェクトのライセンスでライセンスされることに同意し、
CLA（[`CLA.md`](../../../CLA.md)）に署名します。CLA は**再ライセンス権を含む**寛容なライセンスをプロジェクトに付与し、
プロジェクトが BUSL→Apache/MIT の経路を維持し将来ライセンスを調整できるようにします。
