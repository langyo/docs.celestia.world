# モジュール分離と型の所有権

- **Status**: Accepted
- **Date**: 2026-06-09
- **Authors**: Evernight Core Team

## Context

元のコードベースでは、すべてのデータ型が単一の `types.rs` モノリスに配置されていた — 画面、ハードウェア、リモート、プロトコル、トンネルの型が混在していた。これは循環依存のリスクを生み、どのモジュールがどの型を「所有」するかが不明確だった。新しい機能（例: VNC）を追加するには、共有型ファイルの変更が必要だった。

## Decision

各ドメインの型を、それぞれのモジュールの `types.rs` ファイル（例: `screen/types.rs`、`hardware/types.rs`）に移動する。ルートの `types.rs` および `prelude.rs` は後方互換性のためにすべての型を再エクスポートする。各モジュールはその型の唯一の所有者であり、独立して進化できる。

## Consequences

### Positive

- 明確な型の所有権。モジュール間の型の衝突がない
- 新しいモジュールは既存のファイルに触れない

### Negative

- 利用者は正しいモジュールからインポートするか、プレリュードを使用する必要がある
- 再エクスポート層が間接性を追加する
