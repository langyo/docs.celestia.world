# セキュリティポリシー

## 脆弱性の報告

**セキュリティ脆弱性について公開の issue を開かないでください。**

[GitHub Security Advisories](https://github.com/celestia-island/arona/security/advisories/new)
から非公開で報告してください。
GitHub Security Advisories を利用できない場合は、メンテナ宛てに
security@celestia.world へ、明確な説明と再現手順を記載したメールを送信してください。

## 対象範囲

対象内：

- 認証バイパス、JWT/OAuth の弱点、セッション処理の欠陥
- API キー / 認証情報の漏洩や不適切な保管
- 認可と RBAC の実装上の抜け穴
- インジェクション系の脆弱性（SQL、コマンド、SSRF、XSS）
- 安全でないデシリアライズ、パストラバーサル、SSRF
- 権限昇格やテナント間アクセスを許す問題

対象外：

- 本プロジェクトを通じて悪用できない上流依存関係の脆弱性
- 文書化されたガイダンスに反して安全でない構成のセルフホストデプロイ
- 公共 LLM プロバイダのエンドポイントに対するサービス拒否攻撃

## レスポンス

| 段階 | 目標 |
| --- | --- |
| Agent 受領確認 | 10 分以内 |
| 真人受領確認 | 1 暦日以内 |
| 初期評価 | 3 暦日以内 |
| 修正または緩和 | 30 暦日（重要度に依存） |

報告には以下を含めてください：（1）影響を受けるコンポーネントとバージョン、（2）攻撃ベクターと影響、（3）再現手順、（4）提案する緩和策。

## サポート対象バージョン

`main` / `dev` ブランチ上の最新のリリースラインのみがセキュリティ修正の対象となります。
