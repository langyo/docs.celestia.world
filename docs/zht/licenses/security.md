# 安全政策

## 回報漏洞

**請勿針對安全漏洞公開提 issue。**

請透過
[GitHub 安全公告](https://github.com/celestia-island/arona/security/advisories/new)
私下回報。
若您無法使用 GitHub 安全公告，請寄信至維護者
security@celestia.world，並附上清楚的描述與重現步驟。

## 範圍

在範圍內：

- 身分驗證繞過、JWT/OAuth 弱點、工作階段處理缺陷
- API 金鑰 / 憑證洩露或儲存不當
- 授權與 RBAC 實施缺口
- 注入類漏洞（SQL、命令、SSRF、XSS）
- 不安全的反序列化、路徑穿越、SSRF
- 允許提權或跨租戶存取的問題

不在範圍內：

- 上游依賴中無法透過本專案利用的漏洞
- 自架部署未依文件指引進行安全設定
- 針對公共 LLM 提供者端點的阻斷服務攻擊

## 回應

| 階段 | 目標 |
| --- | --- |
| Agent 確認收到 | 10 分鐘內 |
| 真人確認收到 | 1 個日曆日內 |
| 初始評估 | 3 個日曆日內 |
| 修復或緩解 | 30 個日曆日（視嚴重程度而定） |

請在報告中包含：（1）受影響的元件及版本；（2）攻擊向量與影響；（3）重現步驟；（4）建議的緩解措施。

## 支援的版本

僅 `main` / `dev` 分支上的最新發布線會獲得安全修復。
