# 模組解耦與型別所有權

- **狀態**：已接受
- **日期**：2026-06-09
- **作者**：Evernight Core Team

## 背景

原始程式碼庫將所有資料型別放置在一個單一的 `types.rs` 巨石檔案中——混合了螢幕、硬體、遠端、協定和通道型別。這產生了循環相依的風險，並且使得哪個模組「擁有」哪些型別變得不明確。新增功能（例如 VNC）需要修改共享的型別檔案。

## 決策

將每個領域的型別移至其各自模組的 `types.rs` 檔案中（例如 `screen/types.rs`、`hardware/types.rs`）。根目錄的 `types.rs` 和 `prelude.rs` 重新匯出所有型別以保持向下相容。每個模組是其型別的唯一擁有者，可以獨立演進。

## 後果

### 正面

- 清晰的型別所有權；無跨模組型別衝突
- 新模組不會接觸現有檔案

### 負面

- 使用者必須從正確的模組匯入或使用 prelude
- 重新匯出層增加了一層間接性
